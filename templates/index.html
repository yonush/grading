Assessment sub-system

This is a partial application created to present the assessment subsystem for review. This sub-system was created by a different "developer" team as such differences between the main application and this sub-system will be evident.

Features:

The GradeAce assessment sub-system is able to
  • Provide a listing of students
  • Let the user view a students grades
  • Modify assessment grades
  • The grading data is saved to a database
The application is NOT a standalone application and requires integration and dependencies.

Limitations & considerations:

Being a partial application, there are some missing features and design decisions applied;
  • Minimal styling and theme applied - left up to the design team to deal with integration
  • Unit tests have not been included, except for the database driver (schooldb.py)
  • No HTTPS (Hypertext Transfer Protocol Secure) support, no certificates
  • No additional web oriented security & protection (CSRF or XSS)
  • A non multiuser relational database was used for the datastore.
  • Only the bootstrap library for CSS (styling and colours) was used.
  • No session or cookies - the sub-system has no authentication features
  • Rudimentary sanitization and validation conducted on HTTP request & form variables
